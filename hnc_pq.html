Description

Erlang priority queues.

hnc_pq bundles a set of queues, ranked by priority.

For most functions, three flavors exist:
  • Functions addressing the entire priority queue, for example out/1, which will return the item at the front of the highest-priority subqueue which is not empty, if any items exist in the entire priority queue at all.
  • Functions addressing a specific subqueue of the priority queue, for example out/2, which will return the item at the front of the given subqueue, if any items exist in that subqueue at all.
  • Functions addressing a range of subqueues of the priority queue, for example out/3, which will return the item at the front of the highest-priority subqueue in the given range which is not empty, if any items exist in the given range at all.

Functions operate in priority queue order, iterating the subqueues from highest to lowest priority, and processing each subqueue from front to rear (except out_r/1,2,3 which operate in reverse priority queue order).

For this reason, the functions addressing a range of subqueues take the maximum priority as their first and the minimum priority as their second argument, different from what one might otherwise be used to.

Filter items of a priority queue.

The filter function takes two arguments: the priority of each subqueue being filtered in turn, and each item in turn. It must return either true to keep the current item, false to drop it, or a list of items that will be inserted in that place instead of the current item.

The priority queue is filtered from highest to lowest priority, and each subqueue is filtered from front to rear.
